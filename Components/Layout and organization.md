# Layout and organization

- [Boxes](Layout%20and%20organization/Boxes.md) — A box creates a visually distinct group of logically related information and components.
- [Collections](Layout%20and%20organization/Collections.md) — A collection manages an ordered set of content and presents it in a customizable and highly visual layout.
- [Column views](Layout%20and%20organization/Column%20views.md) — A column view — also called a *browser* — lets people view and navigate a data hierarchy using a series of vertical columns.
- [Disclosure controls](Layout%20and%20organization/Disclosure%20controls.md) — Disclosure controls reveal and hide information and functionality related to specific controls or views.
- [Labels](Layout%20and%20organization/Labels.md) — A label is a static piece of text that people can read and often copy, but not edit.
- [Lists and tables](Layout%20and%20organization/Lists%20and%20tables.md) — Lists and tables present data in one or more columns of rows.
- [Lockups](Layout%20and%20organization/Lockups.md) — Lockups combine multiple separate views into a single, interactive unit.
- [Outline views](Layout%20and%20organization/Outline%20views.md) — An outline view presents hierarchical data in a scrolling list of cells that are organized into columns and rows.
- [Split views](Layout%20and%20organization/Split%20views.md) — A split view manages the presentation of multiple adjacent panes of content, each of which can contain a variety of components, including tables, collections, images, and custom views.
- [Tab views](Layout%20and%20organization/Tab%20views.md) — A tab view presents multiple mutually exclusive panes of content in the same area, which people can switch between using a tabbed control.
